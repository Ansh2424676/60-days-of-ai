# Day 9 – NutriScope (Claude Artifacts)

## Overview

For today's challenge, I built an AI-powered Nutrition Analytics application named **NutriScope** using Claude Artifacts. Instead of asking Claude to generate everything in one step, I followed the MVP-first approach and then enhanced the application with a second prompt.

## MVP Version

The first version focused on core functionality:

* User profile inputs
* Food logging
* Nutrition tracking
* Macro and micronutrient calculations
* Dashboard with charts
* Basic food recommendations
* Responsive dark theme interface

## Enhanced Version

The second version expanded the application by adding:

* CSV upload
* Larger food database
* Extra micronutrient tracking
* Two-day meal planner
* Nutrition risk analysis
* Better charts
* Educational disclaimer
* Nutrition sources
* Improved recommendations
* More polished UI and animations

## Comparison

| MVP                      | Enhanced                    |
| ------------------------ | --------------------------- |
| Basic nutrition tracking | Advanced nutrition analysis |
| 20 food items            | 60 food items               |
| Basic dashboard          | Rich interactive dashboard  |
| Simple recommendations   | Smarter recommendations     |
| Standard UI              | Improved visual design      |

## What I Learned

Today's challenge showed me the value of iterative development.

Instead of expecting AI to generate a complex application in a single prompt, creating a working MVP first and then improving it produced a cleaner and more organized result. This approach also made it easier to review and understand the changes introduced in each stage.
